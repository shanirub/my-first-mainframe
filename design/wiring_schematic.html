⚡ PULL-UP RESISTORS — REQUIRED

  • I2C WILL NOT WORK without pull-ups
  • Add 2× 4.7kΩ resistors to your bus
  • One resistor: SDA → 3.3V
  • One resistor: SCL → 3.3V
  • You own resistors already ✅
  • Place near the middle of the bus for best signal

⚠ I2C ADDRESS CONFLICT — IMPORTANT

  • OLED SSD1306 default address: 0x3C
  • Your 5 MCUs use: 0x08–0x0C
  • SOLUTION: Connect each OLED to its OWN MCU's I2C bus (GPIO4/5)
  • Do NOT connect OLEDs to shared bus (GPIO8/9)
  • Each MCU runs TWO I2C instances: one for bus, one for OLED

📋 WIRING ORDER (Step by Step)

  • 1. Place all MCUs on breadboards
  • 2. Run GND rail (black wire) between all MCUs
  • 3. Run 3.3V rail (red wire) between all MCUs
  • 4. Run SDA bus (yellow) between all MCUs
  • 5. Run SCL bus (orange) between all MCUs
  • 6. Add 2× 4.7kΩ pull-ups to bus
  • 7. Connect each OLED to its MCU (GPIO4/5)
  • 8. Connect SD module to MCU #3 (SPI)
  • 9. Connect RTC to MCU #1 (GPIO6/7)
  • 10. Connect USB splitter for power

🔬 LOGIC ANALYZER USAGE

  • CH0 probe → SDA wire on bus
  • CH1 probe → SCL wire on bus
  • GND probe → any GND on breadboard
  • Open PulseView on Fedora
  • Add I2C decoder: CH0=SDA, CH1=SCL
  • Sample rate: 1MHz (plenty for 400kHz I2C)
  • You'll see decoded addresses + data!

📡 TWO I2C BUSES PER MCU

  • Bus 0 — Shared I2C with all MCUs
  • → GPIO8 (SDA), GPIO9 (SCL)
  • → Used for inter-MCU messages
  • Bus 1 — Private OLED connection
  • → GPIO4 (SDA), GPIO5 (SCL)
  • → Used for display only
  • Arduino code: Wire.begin() + Wire1.begin()

🗂 SD CARD MODULE (MCU #3 Only)

  • Uses SPI interface (different from I2C)
  • MOSI → GPIO1
  • MISO → GPIO0
  • SCK → GPIO2
  • CS → GPIO3
  • VCC → 3.3V (module has its own regulator)
  • Format SD card as FAT32 before use

🕐 RTC MODULE (MCU #1 Only)

  • DS1307 uses I2C at address 0x68
  • Connect to MCU #1's private I2C bus
  • SDA → GPIO6 (separate from main bus!)
  • SCL → GPIO7 (separate from main bus!)
  • Install CR2032 battery for time retention
  • Use RTClib library in Arduino

🧪 TEST BEFORE FULL BUILD

  • Week 1: Test 2 MCUs talking via I2C first
  • Run I2C scanner sketch to confirm addresses
  • Test OLED display independently
  • Only add more MCUs once 2-MCU test works
  • Use logic analyzer from day 1!
  • Never assume, always measure!
